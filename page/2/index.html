关于CSRF的一点想法

发表于 2018-12-11 更新于 2019-02-22 分类于 web安全
本文字数： 341 阅读时长 ≈ 1 分钟

刚刚学了一点CSRF漏洞的应用方法，难怪目前都很少有网站存在CSRF漏洞了，由于我们利用CSRF攻击的是其他用户，因此当这些用户出现校验属性的时候，这个漏洞就会显得比较无力，
我们必须与其他漏洞相结合，来获取用户校验属性的值，否则很难构造一个合适的表单，来发送自己想要达成的跨域请求，注入改密码这种敏感操作，各个网站之间都会运用输入原始
密码或者进行邮件校验来防止CSRF攻击，起码目前的CSRF确实很难利用他进行敏感操作了，但起码目前熟悉了CSRF攻击的过程，也熟悉了伪造表单的过程，一些参数的设置或者对


元素的理解，都还是深刻了一些，总之还是觉得有好处的。

我的第一篇文章

发表于 2018-11-20 更新于 2019-02-22
本文字数： 445 阅读时长 ≈ 1 分钟

第一篇博客也是作为一篇测试随便写写的，感觉搭了这么久用next模板做出了这么一个短小精悍的网站还是很有成就感的，就是感觉域名还是有点小贵而已
想了想还是决定目前的方向是往web安全走，但感觉要看的书要学的东西真的好多啊，不过慢慢来嘛，毕竟我才大一。下面贴一点书，既是我自己要读的书，也是作为入门大家可以考虑的一些书籍。
首先做搞前端安全就一定要懂点HTML+Javascript+css，所以感觉《图解HTTP》和《HTML+Javascript+css web开发》是需要学习的，这里涉及到XSS漏洞和CSRF攻击，可以看看《XSS跨站脚本攻击剖析与防御》可以看看
然后搞后端安全就需要对数据库和PHP有一定了解了，所以感觉《PHP基础教程》和《MySQL入门教程》是需要读的，然后就是一本《SQL注入攻击与防御》了，偏向应用一点的方面吧。
搞web感觉计算机网络也是基础吧，所以《计算机网络》也是必要的。
目前就暂时只准备读这些书，感觉实践理论不知道从哪里下手啊。QAQ
